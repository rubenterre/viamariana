# Contexto del Proyecto: Vía Mariana

## 1. Visión General
**Vía Mariana** es un proyecto web desarrollado como parte de las prácticas no laborales del certificado de profesionalidad **IFCD0110** (Confección y publicación de páginas web). El objetivo principal es construir el sitio oficial de la ruta Vía Mariana, aplicando estándares profesionales de desarrollo moderno.

## 2. Objetivos del Proyecto
*   **Diseño y Maquetación:** Crear un sitio web completo que incluya las secciones: `Home`, `Vía Mariana`, `Ruta`, `Patrimonio`, `Proyecto` y `Contacto`.
*   **Consistencia Visual:** Implementar un **Kit de UI** común basado en los wireframes definidos por el equipo.
*   **Flujo de Trabajo Profesional:** Utilizar metodologías de desarrollo colaborativo y despliegue continuo.

## 3. Stack Tecnológico
*   **Framework:** [Astro](https://astro.build) (Generador de sitios estáticos / Componentes).
*   **Lenguajes:** HTML5, CSS3 y JavaScript (estándares modernos).
*   **Diseño:** [Sketch](https://www.sketch.com) para UI, prototipado y wireframes.
*   **Control de Versiones:** Git con repositorio en **GitHub**, utilizando flujo de ramas (feature branches) y Pull Requests.
*   **Despliegue/Hosting:** [Netlify](https://www.netlify.com) con integración de CI/CD.
*   **Gestión:** Notion (Documentación, Kanban y seguimiento de tareas).

## 4. Estructura de Navegación (Sitemap)
1.  **Home:** Introducción visual y resumen del proyecto.
2.  **Vía Mariana:** Historia y concepto de la peregrinación.
3.  **Ruta:** Información detallada sobre las etapas y el recorrido.
4.  **Patrimonio:** Puntos de interés cultural, histórico y natural.
5.  **Proyecto:** Información institucional o del equipo de desarrollo.
6.  **Contacto:** Formulario de comunicación y redes sociales.

## 5. Directrices de Desarrollo para la IA
*   **Componentización:** Al usar Astro, priorizar la creación de componentes reutilizables (Botones, Cards, Hero sections, etc.).
*   **Estilos:** Seguir estrictamente la guía de diseño y el UI Kit definido en Sketch.
*   **Semántica:** Mantener un HTML5 semántico para mejorar el SEO y la accesibilidad (A11y).
*   **Rendimiento:** Aprovechar la isla de arquitectura de Astro para minimizar el JS enviado al cliente.

## 6. Flujo de Trabajo (Git)
*   Las tareas se asignan en el Kanban de Notion.
*   Cada nueva funcionalidad debe desarrollarse en una rama independiente: `feature/nombre-tarea`.
*   Para integrar código, se requiere un Pull Request y revisión previa.
